# congress-trading-shared

Shared TypeScript contracts for the Congress.Trade and Socratic Trade cross-app integration.

This package owns the types, Zod schemas, API path constants, and small utilities that both apps can use for:

- App B -> App A securities imports: `refs`, `prices`, `spx`, `insider`, `shortVolume`, `fundamentals`, `analyst`.
- App A -> App B return-path cache imports: partial refs, prices, and SPX closes.
- Congress.Trade read contracts such as transactions, market refs, bundle responses, analytics rows, and export manifests.

## Commands

```bash
npm install
npm run typecheck
npm run build
npm test
npm audit
npm run pack:dry
```

## Publish Policy

This repo is **public** and the supported consumption path is a **tokenless git
dependency** — no npm registry, no `NODE_AUTH_TOKEN`, no `.npmrc` auth lines.

Consumers pin a package.json dependency directly to this repo, preferring a
semver-stable tag over a moving branch ref:

```json
"@jaywedgeworth22/congress-trading-shared": "github:jaywedgeworth22/congress-trading-shared#semver:^1.2.x"
```

or, if no matching tag exists yet, an exact tag:

```json
"@jaywedgeworth22/congress-trading-shared": "github:jaywedgeworth22/congress-trading-shared#v1.2.0"
```

npm installs the repo tarball at that ref and runs this package's `prepare`
script (`npm run build`), so `dist/` is generated on install — it does not need
to be committed and is not published anywhere. Verify a new pin locally with a
clean, tokenless install before relying on it:

```bash
rm -rf node_modules package-lock.json
npm install github:jaywedgeworth22/congress-trading-shared#<ref>
```

Tag a new release after any change that consumers should pick up:

```bash
git tag vX.Y.Z && git push origin vX.Y.Z
```

(Legacy note: this package briefly published to the private GitHub Packages
registry, `https://npm.pkg.github.com`. That required `NODE_AUTH_TOKEN`/PAT
auth in every consumer and CI job and is retired now that the repo is public —
do not reintroduce a `publishConfig.registry` / scoped-registry `.npmrc` setup
unless the user explicitly asks for a private registry again.)

`dist/` is generated by `npm run build` and omitted from git.

## Contract Notes

- `SecurityRef` is the full read-side shape.
- `SecurityRefInput` is the partial import/upsert shape accepted by the peer import endpoints.
- Date-bearing market/import rows validate as real `YYYY-MM-DD` calendar dates via `IsoDateSchema`.
- `APP_B_ORIGIN` / `APP_B_ORIGIN_TAG` are the no-echo-loop origin tag for Socratic Trade-originated payloads.
