# congress-trading-shared

Shared TypeScript contracts for the Congress.Trade and Agentic Trading cross-app integration.

This package owns the types, Zod schemas, API path constants, and small utilities that both apps can use for:

- App B -> App A securities imports: `refs`, `prices`, `spx`, `insider`, `shortVolume`, `fundamentals`, `analyst`.
- App A -> App B return-path cache imports: partial refs, prices, and SPX closes.
- Congress.Trade read contracts such as transactions, market refs, bundle responses, analytics rows, and export manifests.

## Commands

```bash
npm install
npm run typecheck
npm run build
npm test
npm audit
npm run pack:dry
npm run publish:dry
```

## Publish Policy

This private-repo package publishes to GitHub Packages, not public npm.

- Registry: `https://npm.pkg.github.com`
- Package: `@jaywedgeworth22/congress-trading-shared`
- Release path: create a GitHub release or run the `Publish Package` workflow manually.
- Required permission: `packages: write` via `GITHUB_TOKEN` in the package repo.

Consumers need npm auth for GitHub Packages, for example:

```ini
@jaywedgeworth22:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
```

For local development without registry auth, consume from a git ref or local path.

`dist/` is generated by `npm run build` and omitted from git.

## Contract Notes

- `SecurityRef` is the full read-side shape.
- `SecurityRefInput` is the partial import/upsert shape accepted by the peer import endpoints.
- Date-bearing market/import rows validate as real `YYYY-MM-DD` calendar dates via `IsoDateSchema`.
- `APP_B_ORIGIN` / `APP_B_ORIGIN_TAG` are the no-echo-loop origin tag for Agentic Trading-originated payloads.
